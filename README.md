# chris-thomas-website
Personal website for Chris Thomas, hosted with GitHub Pages.

## Media optimisation

Responsive WebP variants are generated by `vite-imagetools` during development
and production builds. Original image files remain the `src` fallback.

Source MP4 files are discovered automatically in every case-study directory.
Files inside `optimised` directories are ignored. Missing variants—or variants
older than their source—are generated by `npm run optimise:videos`.

Every video requires a manually exported, colour-managed WebP beside the
generated files using the source name followed by `-poster-manual.webp`:

```text
optimised/hero-animation-poster-manual.webp
```

The build reports an error if a manual poster is missing. The optimiser never
creates, changes, or deletes manual poster files.

FFmpeg must be installed locally. Generate the responsive WebM and H.264 MP4
video files directly with:

```bash
npm run optimise:videos
```

Generated videos are written to an `optimised` directory beside each source
video. The source MP4 files are never modified. Commit the generated assets so
production and deployment builds do not require FFmpeg.
