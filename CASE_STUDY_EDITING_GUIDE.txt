Case Study Editing Guide
========================

Case study content and media configuration live in:

src/content/caseStudies.ts

Case study source media lives in:

src/assets/case-studies/

You should not need to edit CaseStudy.tsx, MediaPanel.tsx, or any CSS when
adding or updating normal case study content and media.


Editing Case Study Content
--------------------------

Each entry in the exported caseStudies array controls one homepage tile and
one case study page:

{
  slug: "le-creuset-classic-kettle",
  title: "Le Creuset Classic Kettle.",
  summary: "A short introduction to the project.",
  contextHeading: "Project context.",
  context: "Background, process, constraints, and project goals.",
  role: "Brand design.",
  roleBody: "Responsibilities, collaborators, and deliverables.",
  resultHeading: "Project outcome.",
  resultBody: "Results, impact, and any useful links or metrics.",
  thumbnail: leCreusetMedia[0],
  media: leCreusetMedia,
}

The slug controls the URL. For example:

slug: "le-creuset-classic-kettle"

becomes:

/work/le-creuset-classic-kettle

contextHeading, roleBody, resultHeading, and resultBody are optional. If they
are omitted, the shared defaults in projectContent are used.


Homepage And Navigation Order
-----------------------------

The order of objects in the caseStudies array controls:

- Homepage case study order
- Previous and next case study navigation
- Which project receives each repeating wide or narrow homepage tile layout

The homepage uses the same exported array through homepageCaseStudies. There
is no separate homepage thumbnail order to maintain.

The thumbnail field selects the media displayed on the homepage. It can reuse
an item from the project's media array:

thumbnail: leCreusetMedia[0]

Changing the array index changes the homepage thumbnail without changing the
order of media on the case study page.


Adding A New Case Study Folder
------------------------------

Create a folder beneath:

src/assets/case-studies/

Example:

src/assets/case-studies/my-project/

Keep original source images and videos directly in that project folder. Do not
put source media inside an optimised folder; that folder is reserved for
generated video variants and required manual video posters.

In src/content/caseStudies.ts, add a small path helper:

const myProjectAsset = (file: string) => `my-project/${file}`;

Then define the project's ordered media array:

const myProjectMedia = [
  caseStudyMedia(
    myProjectAsset("my-project-hero.webp"),
    "Describe the hero image for screen-reader users.",
  ),
  caseStudyMedia(
    myProjectAsset("my-project-animation.mp4"),
    "Describe the animated project render.",
  ),
  caseStudyMedia(
    myProjectAsset("my-project-detail.webp"),
    "Describe the project detail image.",
    { layout: "compact" },
  ),
];

Finally, add the new project object to the caseStudies array and select its
thumbnail:

{
  slug: "my-project",
  title: "My Project.",
  summary: "A short introduction to the project.",
  contextHeading: "Project context.",
  context: "Project context copy.",
  role: "My role.",
  roleBody: "Responsibilities and collaborators.",
  resultHeading: "Project outcome.",
  resultBody: "Project results.",
  thumbnail: myProjectMedia[0],
  media: myProjectMedia,
}


Ordering And Sizing Case Study Media
------------------------------------

The order of items in a project's media array controls their order on the case
study page.

- The first item is the lead feature panel.
- Later items use the standard full-width media layout.
- Add { layout: "compact" } to place suitable items in the existing compact
  desktop layout.
- Media defaults to a 16:9 panel. For a differently shaped source, pass its
  real ratio, for example { aspectRatio: "4 / 3" }. Options can be combined as
  { layout: "compact", aspectRatio: "4 / 3" }.
- On mobile, media continues to use the existing responsive page layout.

Do not add wrappers, image elements, video elements, CSS classes, or new CSS
for normal media additions. caseStudyMedia supplies the existing responsive
media implementation while preserving the site's layout and reveal behavior.


Adding Images
-------------

Use WebP for case study images. The current source images are 1920 x 1080.
Keep an image's intended aspect ratio and do not stretch it.

Reference an image through caseStudyMedia:

caseStudyMedia(
  myProjectAsset("my-project-image.webp"),
  "Useful, concise alternative text.",
)

During development and production builds, Vite Image Tools automatically makes
responsive WebP candidates at these widths when the source is large enough:

- 1280
- 1680
- 1920

The original image remains the fallback. Do not manually create, rename, or
place responsive image variants in the source folder.


Adding Videos
-------------

Add the original silent MP4 directly to the project folder and reference it
through caseStudyMedia:

caseStudyMedia(
  myProjectAsset("my-project-animation.mp4"),
  "Useful description of the animation.",
)

The optimisation script discovers source MP4 files automatically. It ignores
optimised folders and creates these video variants beside each source:

optimised/my-project-animation-1280.webm
optimised/my-project-animation-1280.mp4

Create this required poster manually; the optimiser does not create it:

optimised/my-project-animation-poster-manual.webp

The 1280px files are used across screen sizes to remain sharp on high-density
mobile displays. WebM is preferred, with the 1280px MP4 as the compatibility
fallback. Original MP4 files remain local optimisation sources and are not
included in the production bundle.

Generated videos are silent, use the original aspect ratio, and are not
upscaled when the source is smaller than the requested output.

Run npm run optimise:videos after adding or replacing a source video. Missing
or out-of-date files are generated; current files are skipped. FFmpeg must be
installed on the machine running the optimisation.

Before checking a new video in the browser, generate its variants and add its
manual poster:

npm run optimise:videos

Restart the development server after adding newly named media files so Vite
discovers them. Replacing the contents of an existing file does not normally
require a restart.

The generated video variants and manual poster should be committed with the
source video.

Videos display the manual poster immediately and use preload="none". Their
sources are attached only when they approach the viewport. Playback starts
when at least 35% of the video is visible and pauses after it leaves the
viewport. Reduced Motion and Data Saver users retain the static manual poster.


Adding A Manual Video Poster
----------------------------

Every video requires a colour-managed WebP poster created manually. The build
reports an error when the correctly named manual poster is missing.

Place the file in the same optimised folder as the generated video variants.
Its name must exactly match the source video name followed by
-poster-manual.webp.

For this source video:

src/assets/case-studies/my-project/my-project-animation.mp4

use this manual poster path:

src/assets/case-studies/my-project/optimised/my-project-animation-poster-manual.webp

Prefer a 16:9 poster at 1280 x 720 for a 16:9 video. For any other
video shape, preserve the video's real aspect ratio. Export the same opening
frame and match the video's colour-managed output as closely as possible.

The optimiser never creates, changes, or deletes -poster-manual.webp files.


Replacing Existing Media
------------------------

If an image replacement uses the same filename, overwrite the original source
file in the project folder.

If a video replacement uses the same filename, overwrite the original source
MP4 and replace its colour-managed -poster-manual.webp with a matching frame.
Do not manually overwrite generated MP4 or WebM variants.

On the next npm run optimise:videos:

- Responsive image candidates are rebuilt by Vite.
- Video variants are regenerated when the source MP4 is newer than those files.
- The manual -poster-manual.webp file is never changed by the optimiser.

If you rename an original media file, update its caseStudyMedia reference in
src/content/caseStudies.ts. Old generated files with the previous name are not
automatically deleted.


Useful Commands
---------------

Start the local development server:

npm run dev

Generate or refresh video variants:

npm run optimise:videos

Run code and CSS checks:

npm run lint

Run the production build using the committed optimised video assets:

npm run build
